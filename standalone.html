EasyConf Documentation

Quick Start

  1. Paste your network configuration into the left panel
  2. Replace values with variables using {{ `{{ variableName }}` }} syntax
  3. Fill in the variable values on the right panel
  4. Click Generate Configuration
  5. Copy or download your customized configuration

Working with Variables

Variable Syntax

Variables use Jinja2-style syntax with double curly braces:

hostname {{ `{{ hostname }}` }}
interface GigabitEthernet0/0/0
  ip address {{ `{{ ipAddress }}` }} {{ `{{ subnetMask }}` }}

Adding Variables

  • Type {{ `{{ variableName }}` }} directly in your config
  • Or click the + Add Variable button
  • Variables are auto-detected and shown in the right panel
  • Variable names: letters, numbers, underscores only
  • Case-sensitive: {{ `{{ Hostname }}` }} ≠ {{ `{{ hostname }}` }}

Managing Tabs

  • Create: Click the + button in the tab bar
  • Rename: Double-click a tab name to edit it
  • Close: Click the × button (confirms if tab has content)
  • Each tab stores its own config, variables, and output independently

Syntax Highlighting

Choose from the dropdown above the configuration editor:

  • No Highlighting - Plain text (fastest)
  • Bash - For shell scripts
  • Cisco IOS - For Cisco routers/switches
  • Juniper SRX - For Juniper firewalls

Variables are always highlighted in bright orange.

Generating Output

Line Spacing

Use the slider to insert blank lines for readability (1-20 lines, default: 5). Changes apply instantly!

Downloading Output

  • Enter a filename (e.g., router-config.txt)
  • Click the Download button
  • Or use Copy to Clipboard for quick copying

Data Storage & Privacy

  • All data is stored locally in your browser
  • Nothing is sent to any server - 100% private
  • Your tabs, configs, and variables persist across sessions
  • Clearing browser data will delete all saved configurations
  • Works completely offline

Keyboard Shortcuts

  • Tab - Navigate between fields
  • Double-click - Rename tab
  • Enter - Save tab name
  • Esc - Cancel tab rename

Tips & Tricks

  • Use descriptive variable names - {{ `{{ router1Hostname }}` }} is clearer than {{ `{{ h1 }}` }}
  • Create tab templates for different device types
  • Keep backups - Download important configs regularly
  • Test before deploying - Always review generated output
